# Contributing to Heicode Docs

## Pull Requests

All documentation changes should be submitted through Pull Requests.

A PR should include:

- What changed.
- Why the change is needed.
- Verification performed.
- Screenshots when UI changes are involved.

## Public Content Rules

Before publishing:

- Verify the feature exists or is clearly marked as preview.
- Remove private information from screenshots.
- Confirm pricing, limits, and product behavior with the owning team.
- Keep localized documentation synchronized.

## Architecture Changes

Changes that create product or architecture commitments require an ADR review before publication.
