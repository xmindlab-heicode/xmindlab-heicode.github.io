# 创建第一个 Agent

本页说明如何从零创建第一个 Agent 任务。你会完成目标输入、资源连接、角色确认、预算确认和启动。

## 什么时候需要创建 Agent

适合创建 Agent 的场景：

- 从一个产品想法生成 MVP。
- 给现有项目增加功能。
- 修复复杂 bug。
- 生成或整理文档。
- 执行测试和代码审查。
- 部署到云环境。
- 做安全检查或权限整改。

不建议创建 Agent 的场景：

- 目标不明确。
- 没有资源授权。
- 没有验收标准。
- 想绕过人工审批操作生产环境。
- 希望 Agent 长期持有明文密钥。

## 第一步：写清楚目标

好的目标要包含：

- 背景。
- 期望结果。
- 已有资源。
- 不允许做什么。
- 验收标准。

推荐模板：

```text
任务名称：
背景：
要完成的结果：
已有代码或文档：
目标环境：
不允许做的事情：
需要审批的事情：
验收标准：
```

示例：

```text
任务名称：给官网增加法律资源页面
背景：官网需要展示隐私、密钥保管、法律声明。
要完成的结果：生成站内页面和 Markdown 文档。
已有代码或文档：website 目录，docs/legal 目录。
目标环境：Azure Static Web Apps。
不允许做的事情：不要改登录系统，不要暴露密钥。
需要审批的事情：生产部署。
验收标准：页面可访问，文档可被文档站直接使用。
```

## 第二步：确认任务类型

选择任务类型可以帮助 Heicode 推荐角色。

| 任务类型 | 推荐角色 |
|---|---|
| 新产品 | Product、Architect、Frontend、Backend、Reviewer、Ops |
| 功能迭代 | Product、Frontend、Backend、Reviewer |
| Bug 修复 | Debug、Backend、Reviewer |
| 文档 | Docs、Product、Reviewer |
| 部署 | Ops、Reviewer |
| 安全 | Security、Reviewer、Ops |

如果你不确定任务类型，可以让 Heicode 根据目标自动推荐。

## 第三步：选择工作模式

工作模式决定 Agent 怎样拆分任务、怎样协作、什么时候需要你确认。

| 模式 | 适合情况 | 不适合情况 |
|---|---|---|
| Sub 模式 | 子任务边界清楚、多角色分工、复杂排障、代码审查 | 小任务、任务会动态生成、需要自组织协作 |
| 蜂群模式 | 多方向探索、任务动态生成、多 Agent 互相补全、大规模审查 | 预算很紧、验收标准不清楚、生产高危边界不清楚 |
| 瀑布模式 | 需求明确、阶段固定、生产发布、法律或合规内容 | 目标还在探索、需要频繁试错 |
| 敏捷模式 | MVP、产品迭代、UI 调整、多轮反馈 | 无人验收、一次性迁移或强顺序发布 |

推荐选择方式：

- 子任务边界清楚，先选 Sub 模式做可控拆分。
- 任务需要多个 Agent 动态探索和互相补全，选蜂群模式。
- 已经知道要做什么，且风险较高，选瀑布模式。
- 希望先做一版再调整，选敏捷模式。
- 大型任务可以组合使用：敏捷模式管理迭代，每轮内部使用 Sub 或蜂群模式，发布前切换瀑布模式。

详细说明见 [选择 Agent 工作模式](choose-agent-mode.mdx)。

## 第四步：连接 Git 仓库

如果任务需要读写代码，连接 Git 仓库。

操作步骤：

1. 打开任务准备清单。
2. 点击连接 Git。
3. 选择仓库。
4. 选择分支。
5. 设置允许读取的路径。
6. 设置允许写入的路径。
7. 设置是否允许创建分支或 PR。

推荐权限：

| 场景 | 推荐权限 |
|---|---|
| 只分析 | 只读 |
| 小修复 | 当前分支读写 |
| 新功能 | 新分支读写，允许 PR |
| 生产热修 | 读写前必须审批 |

## 第五步：连接文档

文档可以帮助 Agent 理解产品和项目。

建议连接：

- 产品说明。
- 需求文档。
- API 文档。
- 架构文档。
- 设计说明。
- 测试用例。
- 发布说明。

文档默认只读。只有你明确允许时，Docs Agent 才能写入文档目录。

## 第六步：连接 SK 技能库

SK 技能库用于告诉 Agent 团队规范和特定工作方式。

可以包含：

- 编码规范。
- 前端设计规范。
- API 规范。
- 测试规范。
- 部署流程。
- 安全检查清单。
- 文档模板。

建议：

- 只启用当前任务需要的 SK。
- 高危 SK 默认禁用。
- shell、云资源、数据库、部署相关 SK 必须受审批控制。
- 使用固定版本或快照，避免任务过程中规则变化。

## 第七步：连接云资源

如果任务需要部署或访问云资源，连接云账号。

连接后，Heicode 会自动发现可用资源。

你需要确认：

- 哪些资源可以用于本任务。
- 哪些环境可以使用。
- 哪些资源只读。
- 哪些操作需要审批。

生产资源默认不应开放给普通任务。

## 第八步：确认 Agent 角色

Heicode 会推荐子 Agent。

示例：

| 角色 | 本次职责 |
|---|---|
| Product Agent | 整理需求和验收标准 |
| Frontend Agent | 修改页面和交互 |
| Reviewer Agent | 检查 diff 和风险 |
| Ops Agent | 构建和部署 |

你可以减少或增加角色。

建议：

- Sub 模式可以增加多个职责明确的子 Agent。
- 蜂群模式应设置 Agent 角色池、任务领取规则和收敛条件。
- 瀑布模式应按阶段启用角色，不要一开始开放全部角色。
- 敏捷模式应只启用本轮需要的角色。
- 不要创建没有明确职责的角色。
- 不要让同一个 Agent 同时负责开发、审查和部署审批。
- 不要给所有角色相同权限。

## 第九步：确认预算

预算用于控制模型消耗。

需要确认：

- 最大 token。
- 最大成本。
- 最大执行时间。
- 超额是否暂停。
- 大额消耗是否审批。

建议：

| 任务 | 建议方式 |
|---|---|
| 小修复 | 小预算，超出暂停 |
| 中等功能 | 分阶段预算 |
| MVP | 阶段验收后继续 |
| 文档 | 低预算，限制工具 |
| 部署 | 小预算，高审批 |
| Sub 模式 | 按子 Agent 设置预算，异常子任务可单独暂停 |
| 蜂群模式 | 设置总预算、单 Agent 上限、最大轮次和收敛阈值 |
| 瀑布模式 | 按阶段设置预算，阶段完成后再释放下一段 |
| 敏捷模式 | 按迭代设置预算，每轮验收后再继续 |

## 第十步：确认审批规则

必须审批的操作：

- 生产部署。
- 生产数据库读写。
- 访问生产密钥。
- 删除云资源。
- 扩缩容。
- 外部发送敏感数据。
- 合并到主分支。
- 大额模型消耗。

审批时你会看到：

- 谁请求。
- 哪个 Agent 请求。
- 要操作什么资源。
- 风险等级。
- 预计影响。
- 凭证有效期。
- 是否可回滚。

## 第十一步：启动 Agent

启动前确认：

- 目标清楚。
- 工作模式已选择。
- 资源已连接。
- 角色合理。
- 权限最小。
- 预算已设置。
- 审批规则已设置。
- 验收标准明确。

点击启动后，Agent 会进入任务视图。

你可以关闭页面后稍后回来查看。任务状态会保留在 Heicode 中。
