# 监控、交付和继续迭代

启动 Agent 后，你不需要一直盯着完整日志。任务视图会告诉你当前进度，以及什么时候需要你输入、审批或验收。

## 监控什么

Manager 中应重点看：

- 工作模式。
- 任务状态。
- 当前阶段。
- 子 Agent 状态。
- 最近事件。
- 待审批项。
- 失败原因。
- 模型用量。
- 预算剩余。
- 资源访问记录。
- 交付物。

## 按工作模式监控

### Sub 模式

重点看：

- 子 Agent 是否都在做不同事情。
- 是否存在重复执行。
- 是否有子任务长时间阻塞。
- 是否有结果互相矛盾。
- 哪个子 Agent 消耗最多预算。
- 主 Agent 是否已经聚合阶段结果。

### 蜂群模式

重点看：

- 共享任务池是否不断膨胀。
- 是否有多个 Agent 重复处理同一任务。
- 是否出现循环 Handoff。
- 共享状态是否出现冲突。
- 是否迟迟无法收敛。
- 哪些 Agent 或任务类型消耗最多预算。
- 高危操作是否退出自动流程并进入审批。

### 瀑布模式

重点看：

- 当前阶段是否完成。
- 阶段输出是否满足进入下一阶段的条件。
- 是否有未经确认就进入下一阶段。
- 发布、生产、密钥和主分支操作是否已经审批。
- 回滚方案是否完整。

### 敏捷模式

重点看：

- 本轮目标是否仍然清楚。
- 本轮范围是否被扩大。
- 本轮交付物是否可验收。
- 反馈是否进入下一轮，而不是混入当前轮。
- 连续多轮变更是否说明目标需要重新定义。

## 执行阶段

常见阶段：

```text
理解目标
-> 准备上下文
-> 生成计划
-> 执行修改
-> 运行验证
-> 修复问题
-> 生成交付说明
-> 等待验收
-> 可选部署
```

不同任务阶段会不同。文档任务可能没有部署阶段，部署任务可能没有代码实现阶段。

## 查看日志

日志用于理解 Agent 做了什么。

你可以查看：

- 当前动作。
- 工具调用摘要。
- 测试结果。
- 构建结果。
- 错误原因。
- 部署日志。
- 审计事件。

日志不应包含明文密钥。如果看到疑似密钥，应立即停止任务并通知管理员。

## 查看模型用量

用量页面应展示：

- 本任务消耗。
- 子 Agent 消耗。
- 今日消耗。
- 剩余额度。
- 高成本调用。
- 失败重试消耗。

如果消耗异常：

- 暂停任务。
- 查看是否循环失败。
- 缩小任务范围。
- 降低模型成本。
- 拆分任务。

## 查看交付物

交付物可能包括：

- 需求摘要。
- 技术方案。
- 代码 diff。
- PR。
- 测试报告。
- 构建记录。
- 部署地址。
- 发布说明。
- 文档。
- 审计摘要。

## 验收结果

建议使用四种结论：

| 结论 | 含义 |
|---|---|
| 通过 | 满足验收标准 |
| 有条件通过 | 基本可用，但有明确后续项 |
| 退回修改 | 不满足要求，需要继续迭代 |
| 停止任务 | 目标变化或风险过高 |

## 继续迭代

你可以在客户端继续输入：

- 修改某个功能。
- 增加新功能。
- 修复测试失败。
- 调整样式。
- 优化性能。
- 生成文档。
- 部署到另一个环境。

继续迭代时，Heicode 会复用已有上下文、资源和审计记录。

## 什么时候新建任务

建议新建任务的情况：

- 目标完全不同。
- 资源范围不同。
- 需要不同审批人。
- 需要不同预算。
- 上一个任务已经交付并归档。

## 什么时候停止任务

建议停止任务的情况：

- 方向错误。
- 权限配置错误。
- 预算失控。
- 资源授权错误。
- 出现安全风险。
- 任务无法继续。

停止后先检查审计和临时资源，再决定是否复制任务重新开始。
