# 快速开始

本页用一个最短路径说明如何在 Heicode 中启动一个 Agent 任务。你不需要提前配置模型供应商，也不需要接触密钥保管器后台。

## 开始前你需要准备

- 一个 Heicode 账号。
- 已安装或可下载 Heicode 客户端。
- 一个要完成的目标，例如修复 bug、开发功能、生成文档或部署项目。
- 如果任务需要代码，准备一个 Git 仓库。
- 如果任务需要部署，准备一个云账号或可授权的云资源。

## 第一步：登录 Heicode

1. 打开 Heicode Manager。
2. 使用你的账号登录。
3. 确认首页能看到任务、资源、模型用量和客户端下载入口。![31f865c364999514ec5299004425768c.png](/screenshots/quickstart-signin.png)
## 第二步： 资源绑定
点击左侧资源绑定，Git资源是基础，sub和swarm蜂群模式都以git资源绑定后才能使用。
常见资源包括：
- Git 仓库。
- 产品文档。
- SK 技能库。
- 云账号。
- 云资源，例如数据库、静态站点、Kubernetes、对象存储。
![2a669e9121f0e93b8d4cd8b139fc6de0.png](/screenshots/quickstart-bind-resources.png)
连接资源后，Heicode 会展示脱敏状态和资源范围。长期密钥会进入密钥保管器，页面只展示引用和状态。


## 第三步：部署Sub  Agent
在页面左侧点击部署Sub  Agent，可以选择单Agent以及开发者团队，部署好了之后在页面下方同步显示部署的Agent（默认最多安装 5 个 Agent，具体数量以套餐为准）。![8e97aca64a191270b2569c4e3584d388.png](/screenshots/quickstart-deploy-agent-1.png)![4cbf20e9f399e8e0a1b2502d9c3da294.png](/screenshots/quickstart-deploy-agent-2.png)更换Agent以及删除已经部署好的Agent都在这个位置。![595d9d915c1e665d3fe3ad82e0f46ba5.png](/screenshots/quickstart-deploy-agent-3.png)
## 第四步：下载安装Heicode客户端
在页面左侧点击Heicode桌面客户端，下载对应系统客户端并打开。![87ca1f402d1c30dc21b096fccfa3a4dc.png](/screenshots/quickstart-client-download.png)

## 第五步：调用部署的Sub  Agent
在客户端中完成登录。点击客户端主页面中的Agents，可以看见并选择您刚在Heicode Manager中部署的Agent，勾选即可开始使用。![4e61204890ef30f44bf432a88758f49c.png](/screenshots/quickstart-call-agent.png)

用户主要在客户端里输入任务和处理审批。Manager 用于资源准备、启动 Agent、查看状态、用量和审计。

## 第六步：输入一个目标

在客户端输入你要完成的事情。

推荐格式：

```text
我要做什么：
已有代码或文档：
希望部署到哪里：
哪些操作必须先问我：
验收标准：
```

示例：

```text
我要为现有官网增加一个资源中心页面。
已有 docs/product-package 资料。
资源中心要写成站内文档，不要跳转到 Git。
构建通过后部署到 Azure Static Web Apps。
部署前需要我确认。
```

Heicode 会根据目标生成需求摘要、任务计划、资源建议和推荐 Agent 角色。你只需要按清单补齐缺失项，不需要填写底层部署参数。
## 第七步：确认 Agent 推荐摘要

Heicode 会生成一个你能看懂的摘要。

示例：

```text
本任务建议启动：
- Product Agent：整理需求和验收标准
- Frontend Agent：实现页面
- Reviewer Agent：检查改动
- Ops Agent：准备部署

将使用：
- 仓库：website
- 文档：docs/product-package
- 云资源：Azure Static Web Apps

需要审批：
- 生产部署
- 大额模型消耗
- 访问生产密钥
```
![aa52eb774210b84cc7cdaa855f1493e3.png](/screenshots/quickstart-recommendation-summary.png)

确认摘要后，点击启动 Agent。

## 第八步：观察任务状态

任务启动后，你会看到任务视图。

任务只有四种真实状态：

| 状态 | 你应该做什么 |
|---|---|
| pending（准备中） | 等待平台整理资源和上下文 |
| running（运行中） | 等待 Agent 执行，可查看进度 |
| stopped（已停止） | 任务被你或平台停止 |
| failed（失败） | 查看失败原因并重试或调整资源 |

除了这四种状态，任务视图里出现的“需要输入”“等待审批”“可验收”，其实都是任务处于 running 时的具体情况，不是单独的顶层状态：

- 需要输入：Agent 触发了 `task.blocked` 事件，缺少信息、资源或权限，需要你打开任务补充。
- 等待审批：出现了一条待处理的审批请求（审批请求本身有自己的状态机：pending / approved / rejected / expired），需要你查看风险后批准或拒绝。
- 可验收：任务仍在 running，但已经产出可检查的交付物，你可以提前查看结果。

任务顺利跑完后，状态会变为 stopped（正常结束），同时会附带一条 `task.completed` 事件和交付物，界面上通常展示为“已完成”，与你主动中止的 stopped 加以区分。

## 第九步：处理需要你介入的事项

Agent 可能会在这些情况下需要你：

- 需求不清楚。
- 资源没有连接。
- 权限不足。
- 预算不足。
- 需要生产部署。
- 需要访问生产密钥。
- 需要你确认实现方向。
![761778426bec6c7808f2fc35593e82c2.png](/screenshots/quickstart-task-attention.png)

你可以在任务视图中回复、审批、拒绝、暂停或停止任务。

## 第十步：查看结果

任务完成后，检查：

- 交付物。
- 代码 diff。
- 测试结果。
- 构建结果。
- 部署结果。
- 模型用量。
- 审批记录。
- 审计记录。
![ffb31c1ec3f1008c1097730aed75b3f7.png](/screenshots/quickstart-review-results.png)
如果结果不完整，可以继续在客户端补充要求，让 Agent 进入下一轮迭代。
