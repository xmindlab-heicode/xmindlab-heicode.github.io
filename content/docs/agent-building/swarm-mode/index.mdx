# 蜂群模式

Heicode 的蜂群模式采用 lead（主管）加 worker（工作）的主从架构：每个租户对应一台 Azure 虚拟机，虚拟机上以 docker-compose 运行一个 api 容器、一个 lead 容器和若干 worker 容器。lead 负责接收目标、拆解任务并派发给 worker；worker 负责领取任务、执行并上报心跳，最终由 lead 聚合结果。worker 数量最多 8 个，默认 3 个（随套餐可调整）。

这不是没有中心调度、Agent 自行竞争任务的去中心化系统，而是有明确 lead 负责拆解和调度的主从架构。

本节包含：蜂群特色说明、协作模式对比、Sub 模式结合瀑布/敏捷的用法，以及蜂群能力评价标准（目标框架）。
