# 理解任务视图和状态

Agent 任务视图用于集中查看多个 Agent 任务。它的目标是让你一眼知道：哪些任务正在工作，哪些任务需要你输入，哪些任务已经完成，哪些任务失败了。

这类任务视图的设计参考了 Claude Code Agent View 的客户体验：先让用户看到状态，再在需要时打开详情、回复、审批或接管。

## 任务视图显示什么

每个任务行通常包含：

- 任务名称。
- 工作模式。
- 当前状态。
- 当前正在做什么。
- 最后更新时间。
- 负责的 Agent 角色。
- 是否需要用户输入。
- 是否有待审批操作。
- 是否有交付结果。
- 是否有失败原因。
- 模型消耗和预算状态。

## 工作模式会影响你看到什么

不同工作模式下，任务视图展示重点不同。

| 模式 | 任务视图重点 | 用户最常处理的动作 |
|---|---|---|
| Sub 模式 | 子 Agent 状态、阻塞项、显式交接、结果聚合 | 停止跑偏子任务、补充上下文、审批高危动作 |
| 蜂群模式 | lead 的任务拆解情况、各 worker 的领取和执行状态、结果聚合 | 处理阻塞的 worker、调整预算或接管 |
| 瀑布模式（5 角色固定模板） | 当前阶段、阶段输出、阶段确认、发布审批 | 确认阶段结果、退回上一阶段、批准发布 |
| 敏捷模式（3 角色固定模板） | 本轮目标、本轮交付、反馈项、下一轮建议 | 验收增量、调整优先级、开始下一轮 |

## 常见状态

任务（deployment）本身只有四种真实状态：

| 状态 | 含义 | 用户动作 |
|---|---|---|
| pending（准备中） | Heicode 正在整理资源和上下文 | 等待 |
| running（运行中） | Agent 正在执行任务 | 可查看进度 |
| stopped（已停止 / 已完成） | 用户或平台停止任务，包括正常跑完的情况 | 查看结果、重启或复制任务 |
| failed（已失败） | 任务因错误停止 | 查看失败原因 |

“需要输入”“等待审批”“可验收”“已阻塞”不是独立的顶层任务状态，而是任务处于 running 时的具体情况，通常对应下面这些真实信号：

| 你在界面上看到的说法 | 对应的真实信号 | 用户动作 |
|---|---|---|
| 需要输入 / 已阻塞 | 任务仍是 running，Agent 触发了 `task.blocked` 事件 | 打开任务并回复 |
| 等待审批 | 任务仍是 running，存在一条状态为 pending 的审批请求 | 查看风险并批准或拒绝 |
| 可验收 | 任务仍是 running 或刚变为 stopped，已经产出可检查的交付物 | 查看交付物 |

审批请求是一个独立的对象，有自己的状态机：`pending`（待处理）、`approved`（已批准）、`rejected`（已拒绝）、`expired`（已过期），不属于任务状态本身。

## 状态优先级

任务视图应优先展示：

1. 存在待处理审批请求的任务（俗称“等待审批”）。
2. 触发 `task.blocked` 的任务（俗称“需要输入”）。
3. 已产出交付物、等待你检查的任务（俗称“可验收”）。
4. failed。
5. running（其他情况）。
6. stopped（正常完成）。

这样用户不需要在大量日志中寻找需要处理的事项。

## 查看任务详情

打开任务后，你可以看到：

- 目标摘要。
- 当前阶段。
- Agent 角色。
- 最近动作。
- 关键日志。
- 资源使用记录。
- 模型消耗。
- 待审批项。
- 交付物。
- 失败原因。

## 回复 Agent

当任务显示“需要输入”时，你可以回复：

- 补充需求。
- 修改优先级。
- 选择方案。
- 上传或连接更多文档。
- 调整资源范围。
- 要求先生成计划。
- 要求停止某个方向。

回复应尽量具体。

不推荐：

```text
继续。
你看着办。
快点。
```

推荐：

```text
先只实现用户登录和项目列表，不要做通知模块。
后端只改 api/projects 相关接口。
不要访问生产数据库。
完成后先给我测试结果，不要部署。
```

## 查看交付结果

交付结果可能包括：

- 需求文档。
- 技术方案。
- 代码 diff。
- 测试结果。
- 构建结果。
- 部署日志。
- 运行地址。
- 审计记录。
- 后续建议。

验收前不要只看最终一句话，应检查关键产物。

## 接管任务

当任务复杂或需要人工判断时，可以接管。

接管后你可以：

- 查看完整上下文。
- 直接回复更详细要求。
- 调整资源授权。
- 拆分任务。
- 要求重新规划。
- 停止当前执行。

接管不等于放弃任务。它只是让你从概览视图进入更详细的任务对话。

## 停止任务

可以停止任务的情况：

- 目标写错。
- 资源选错。
- 预算异常。
- Agent 方向明显错误。
- 任务卡住。
- 出现安全风险。
- 不再需要执行。

停止任务后应检查：

- 是否有未提交代码。
- 是否有临时资源。
- 是否有未关闭部署。
- 是否有仍有效的短期凭证。
- 是否需要保留审计。

## 重新运行任务

重新运行前建议先调整：

- 目标描述。
- 资源范围。
- Agent 角色。
- 预算。
- 审批规则。
- SK 技能。

不要在同样错误条件下反复重试，否则会浪费模型预算。
