# Team FAQ and Troubleshooting

This page is for team administrators and task owners.

## A member can't see a task

Possible causes:

- Not part of the organization.
- No project permission.
- The task belongs to another team.
- Logged in with the wrong account.

Handling:

- Check the organization's member list.
- Check the member's role.
- Check the task's visibility scope.
- Have the member log in again.

## A member can't approve

Possible causes:

- Not an approver.
- Approval permission has been revoked.
- The approval has expired.
- The operation requires a higher level of approval.

Handling:

- Check the approval rules.
- Add a backup approver.
- Re-initiate the approval.
- Have the Owner or Admin adjust permissions.

## Agent can't access the repository

Possible causes:

- Git authorization has expired.
- Insufficient repository permissions.
- The branch does not exist.
- The path scope is too narrow.

Handling:

- Re-authorize Git.
- Check repository permissions.
- Adjust the path scope.
- Restart the task.

## Too many task failures

Possible causes:

- The task is too large.
- The goal is unclear.
- Insufficient resources.
- Insufficient permissions.
- The test environment is unstable.

Handling:

- Split the task.
- Add acceptance criteria.
- Add resources.
- Adjust permissions.
- Have the Agent generate a plan first before executing.

## No one is handling an approval

Handling:

- Check whether the approver is online.
- Check whether the approver is still in the organization.
- Set a backup approver.
- Re-initiate an expired approval.
- Escalate urgent matters to the Owner.

## Abnormal budget consumption

Handling:

- Pause the task.
- Check sub-agent consumption.
- Check for failure loops.
- Reduce model cost.
- Split the task.
- Set a lower budget threshold.

## Production deployment failure

Handling:

1. Check the deployment logs.
2. Check environment variables.
3. Check secret references.
4. Check cloud resource status.
5. Check approval records.
6. Execute a rollback.
7. Create a fix task.

## Suspected secret leak

Handle immediately:

1. Stop the related task.
2. Revoke credentials.
3. Rotate secrets.
4. Check Git, logs, Markdown, and screenshots.
5. Notify Security.
6. Review the audit records.
7. Fix the exposed location.

## The team doesn't know what to do next

It is recommended to check three places:

- Task status.
- Readiness checklist.
- Pending approval list.

If it is still unclear, have the Agent generate a next-step plan first — do not directly execute high-risk actions.
