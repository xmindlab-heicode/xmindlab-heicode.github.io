# 团队快速开始

<Callout type="warn" title="当前状态说明">
  组织、成员邀请、角色分配等团队管理能力目前仍在规划/后端阶段，尚无独立的团队管理界面。本页中的审批、密钥托管、审计相关描述是已在单任务/单 Agent 场景下真实运行的机制；涉及多成员组织和团队角色的部分为面向未来的使用建议。
</Callout>

本页说明一个团队第一次使用 Heicode 时应该怎么开始。

## 第一步：创建团队或加入团队

团队管理员创建组织后，邀请成员加入。

成员会收到邀请，然后：

1. 使用自己的账号登录。
2. 确认组织名称。
3. 检查自己的角色。
4. 下载 Heicode 客户端。
5. 查看团队任务列表。

不要多人共享同一个账号。

## 第二步：设置基础角色

Heicode 组织当前的真实角色只有 4 个：

- Owner：负责组织、套餐、账务和最终权限。
- Admin：负责成员、资源、任务和审计。
- Member：参与任务的普通成员。
- Auditor：只读查看审计记录。

小团队可以一人兼多角色，但权限要清楚。技术方案、需求验收、部署运维、安全策略等分工目前是团队内部的职责约定，不是平台上可选择的组织角色。

Agent 角色是另一套体系，与上面的组织角色无关，包括 Product Agent、Architect Agent、Frontend Agent、Backend Agent、Reviewer Agent、Ops Agent 六种，详见[角色和权限](/docs/team-usage/roles-and-permissions)。

## 第三步：连接基础资源

管理员或负责人连接：

- Git 仓库。
- 产品文档。
- SK 技能库。
- 云账号。
- 部署资源。

连接后，团队应确认：

- 谁可以使用这个资源。
- 哪些任务可以使用这个资源。
- 是否只读。
- 是否允许写入。
- 是否是生产资源。
- 哪些操作需要审批。

## 第四步：设置默认审批规则

建议默认审批：

- 生产部署。
- 生产数据库读写。
- 生产密钥访问。
- 云资源删除。
- 主分支合并。
- 大额模型消耗。

每个高危操作可以指定一个目标审批角色（`TargetRole`），由该角色的成员批准。具体哪个操作配哪个审批角色，目前由任务发起时的实际设置决定，不是平台预置的固定规则表，团队可以按自己的分工习惯约定，例如生产部署、生产数据库、生产密钥优先由熟悉运维和安全的成员审批，大额预算优先由 Owner 审批。

## 第五步：创建第一个团队任务

建议从低风险任务开始，例如：

- 生成文档。
- 修复小 bug。
- 增加非生产页面。
- 运行测试并整理报告。

不要一开始就让 Agent 直接操作生产资源。

## 第六步：启动 Agent

任务负责人确认：

- 目标清楚。
- 仓库正确。
- 资源范围正确。
- Agent 角色合理。
- 预算合理。
- 高危审批规则已设置。

然后启动 Agent。

## 第七步：团队处理任务状态

任务启动后，团队主要关注：

- 是否需要输入。
- 是否等待审批。
- 是否失败。
- 是否可验收。
- 是否消耗异常。

团队不需要每个人都看完整日志。负责人看任务状态，相关角色在需要时介入。

## 第八步：验收和复盘

完成后检查：

- 是否满足验收标准。
- 是否有测试结果。
- 是否有代码审查。
- 是否有审批记录。
- 是否有部署记录。
- 是否有成本记录。

第一次任务完成后，团队应复盘：

- 任务描述是否清楚。
- 资源授权是否合适。
- 审批人是否正确。
- Agent 角色是否太多或太少。
- 预算是否合理。
