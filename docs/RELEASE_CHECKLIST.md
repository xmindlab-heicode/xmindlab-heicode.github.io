# Public Documentation Release Checklist

Before publishing changes to docs.heicode.cc:

## Content

- [ ] Content is user-facing and approved for public release
- [ ] No internal architecture details are exposed
- [ ] No credentials, tokens, customer data, or private infrastructure information are included
- [ ] Screenshots are reviewed and sanitized

## Product accuracy

- [ ] Product behavior matches the released version
- [ ] Version compatibility is documented where needed
- [ ] Deprecated or preview features are clearly labeled

## Localization

- [ ] English documentation updated
- [ ] Chinese documentation updated
- [ ] Navigation and links are consistent

## Review

- [ ] Required owner review completed
- [ ] Architecture changes have an ADR reference
- [ ] Security-sensitive changes have Security review
