\--- title: 快速开始 description: 用最短路径创建并运行第一个 Agnet 任务。 updated: 2026-05-14 \--- # 快速开始 本页用一个最短路径说明如何在 Heicode 中启动一个 Agnet 任务。你不需要提前配置模型供应商，也不需要接触密钥保管器后台。 ## 开始前你需要准备 \- 一个 Heicode 账号。 \- 已安装或可下载 Heicode 客户端。 \- 一个要完成的目标，例如修复 bug、开发功能、生成文档或部署项目。 \- 如果任务需要代码，准备一个 Git 仓库。 \- 如果任务需要部署，准备一个云账号或可授权的云资源。 ## 第一步：登录 Heicode 1\. 打开 Heicode Manager。 2\. 使用你的账号登录。 3\. 确认首页能看到任务、资源、模型用量和客户端下载入口。 4\. 下载并打开 Heicode 客户端。 5\. 在客户端中完成登录。 用户主要在客户端里输入任务和处理审批。Manager 用于资源准备、启动 Agnet、查看状态、用量和审计。 ## 第二步：输入一个目标 在客户端输入你要完成的事情。 推荐格式： ```text 我要做什么： 已有代码或文档： 希望部署到哪里： 哪些操作必须先问我： 验收标准： ``` 示例： ```text 我要为现有官网增加一个资源中心页面。 已有 docs/product-package 资料。 资源中心要写成站内文档，不要跳转到 Git。 构建通过后部署到 Azure Static Web Apps。 部署前需要我确认。 ``` Heicode 会根据目标生成需求摘要、任务计划、资源建议和推荐 Agnet 角色。 ## 第三步：打开准备清单 进入 Manager 中当前任务的准备清单。 准备清单会告诉你： \- 是否需要连接 Git。 \- 是否需要连接文档。 \- 是否需要连接 SK 技能库。 \- 是否需要连接云账号。 \- 是否需要设置预算。 \- 是否需要设置审批人。 你只需要按清单补齐缺失项，不需要填写底层部署参数。 ## 第四步：连接资源 常见资源包括： \- Git 仓库。 \- 产品文档。 \- SK 技能库。 \- 云账号。 \- 云资源，例如数据库、静态站点、Kubernetes、对象存储。 连接资源后，Heicode 会展示脱敏状态和资源范围。长期密钥会进入密钥保管器，页面只展示引用和状态。 ## 第五步：确认 Agnet 推荐摘要 Heicode 会生成一个你能看懂的摘要。 示例： ```text 本任务建议启动： \- Product Agnet：整理需求和验收标准 \- Frontend Agnet：实现页面 \- Reviewer Agnet：检查改动 \- Ops Agnet：准备部署 将使用： \- 仓库：website \- 文档：docs/product-package \- 云资源：Azure Static Web Apps 需要审批： \- 生产部署 \- 大额模型消耗 \- 访问生产密钥 ``` 确认摘要后，点击启动 Agnet。 ## 第六步：观察任务状态 任务启动后，你会看到任务视图。 常见状态： | 状态 | 你应该做什么 | |---|---| | 准备中 | 等待平台整理资源和上下文 | | 运行中 | 等待 Agnet 执行 | | 需要输入 | 打开任务，回答问题或补充信息 | | 等待审批 | 查看风险后批准或拒绝 | | 已完成 | 查看交付物和审计 | | 失败 | 查看失败原因并重试或调整资源 | | 已停止 | 任务被你或平台停止 | ## 第七步：处理需要你介入的事项 Agnet 可能会在这些情况下需要你： \- 需求不清楚。 \- 资源没有连接。 \- 权限不足。 \- 预算不足。 \- 需要生产部署。 \- 需要访问生产密钥。 \- 需要你确认实现方向。 你可以在任务视图中回复、审批、拒绝、暂停或停止任务。 ## 第八步：查看结果 任务完成后，检查： \- 交付物。 \- 代码 diff。 \- 测试结果。 \- 构建结果。 \- 部署结果。 \- 模型用量。 \- 审批记录。 \- 审计记录。 如果结果不完整，可以继续在客户端补充要求，让 Agnet 进入下一轮迭代。
