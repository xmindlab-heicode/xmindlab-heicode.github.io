# 仓库与文档同步清单

本文用于说明 xmindlab-heicode 团队当前有哪些主要项目仓库、各自承担什么职责，以及当这些仓库出现新功能、界面变化、代码调整或发布更新时，文档仓库中通常需要同步更新哪些页面。

## 使用原则

- 产品事实优先来自代码主线仓库，而不是反向以文档仓库作为事实来源。
- 涉及页面流程、接口能力、资源绑定、Agent 部署、审计、模型和权限的变化，优先核对 `heicode-mananger`。
- 涉及客户端交互、登录体验、版本变化、错误提示、模型选择、任务视图等变化，优先核对 `heicode-macos-release-dev` 和 `heicode-winos-release-dev`。
- 涉及单 Agent、Sub 模式、常驻 Agent、部署管理和运行编排，优先核对 `agent_management`。
- 涉及蜂群模式、Swarm 控制面、审批流、租户运行时和功能面对接，优先核对 `agent_swarm`。
- 涉及 SK 技能库结构、技能安装、权限声明和技能注册方式，优先核对 `heicode-sk-hub`。
- `heicode-macos-release`、`heicode-winos-release` 主要用于发布校验；`heicodeDocs` 和 `xmindlab-heicode.github.io` 属于文档输出侧，不应反向作为产品事实源。

## 当前团队项目仓库有哪些，分别是什么

下面按当前团队组织仓库整理，说明每个仓库分别承担什么职责，以及文档同步时应如何看待它。

### 1. `heicode-mananger`

- **类型**：平台后台 / 管理控制台
- **分别是什么**：Heicode Manager 主线仓库，负责模型网关、管理控制台、产品共识、集成契约和部分产品包文档。
- **更新信号**：平台入口、资源绑定、模型用量、审计、下载入口、账号体系、后端集成契约变化，通常优先从这里判断。
- **文档同步定位**：平台登录、资源绑定、模型余额、审计、下载入口等事实优先以本仓为准。
- **是否为主要文档事实源**：是

### 2. `heicode-macos-release-dev`

- **类型**：macOS 客户端开发
- **分别是什么**：macOS 客户端开发主线，决定桌面端功能、交互、登录体验、错误提示和版本能力。
- **更新信号**：客户端界面、交互、登录方式、任务视图、模型列表、审批体验、错误提示、自动更新变化，优先看本仓。
- **文档同步定位**：客户端功能、任务视图、错误提示和版本能力更新时优先核对本仓。
- **是否为主要文档事实源**：是

### 3. `heicode-winos-release-dev`

- **类型**：Windows 客户端开发
- **分别是什么**：Windows 客户端开发主线，决定 Windows 端功能、兼容性、路径处理、权限切换和版本能力。
- **更新信号**：Windows 端特有交互、稳定性、权限模式、路径兼容、升级行为变化时，优先看本仓。
- **文档同步定位**：Windows 端功能、兼容性和版本变化应优先核对本仓。
- **是否为主要文档事实源**：是

### 4. `agent_management`

- **类型**：Agent 管理平台
- **分别是什么**：单 Agent、Sub 模式、常驻 Agent 的部署、管理和运行编排主线。
- **更新信号**：部署入口、部署数量限制、Agent 生命周期、常驻 Agent 管理、团队部署方式变化时，优先看本仓。
- **文档同步定位**：部署入口、部署上限、生命周期状态等事实应优先看本仓。
- **是否为主要文档事实源**：是

### 5. `agent_swarm`

- **类型**：蜂群运行时
- **分别是什么**：蜂群模式、Swarm Controller、多租户运行时、审批和客户端蜂群对接主线。
- **更新信号**：蜂群能力、Swarm 控制面 / 功能面对接、审批链路、租户运行方式、蜂群能力边界变化时，优先看本仓。
- **文档同步定位**：蜂群能力、控制面 / 功能面对接和审批链路应优先看本仓。
- **是否为主要文档事实源**：是

### 6. `heicode-sk-hub`

- **类型**：SK 技能库
- **分别是什么**：官方 SK 技能仓库，负责技能注册、元数据、权限声明和安装来源。
- **更新信号**：SK 技能包结构、注册方式、权限声明、技能安装与引用方式变化时，优先看本仓。
- **文档同步定位**：SK 结构、权限和注册方式应优先以本仓为准。
- **是否为主要文档事实源**：是

### 7. `heicode-webhtml`

- **类型**：Web 前端工程
- **分别是什么**：体量较大的 Web 前端工程仓库，可作为页面实现和界面细节的补充参考。
- **更新信号**：当页面视觉、导航细节、前端交互变化无法从 `heicode-mananger` 明确判断时，可补充查看本仓。
- **文档同步定位**：当前不应替代 `heicode-mananger` 作为平台事实主源，仅在需要核对前端细节时补充参考。
- **是否为主要文档事实源**：视情况补充参考

### 8. `heicode-macos-release`

- **类型**：macOS 发布仓库
- **分别是什么**：macOS 客户端打包、签名、公证和发布自动化仓库。
- **更新信号**：安装包、公证、签名、分发方式或最终外部版本号变化时，用于发布校验。
- **文档同步定位**：主要用于校验最终外部版本、安装包和分发状态。
- **是否为主要文档事实源**：否，作为发布校验源

### 9. `heicode-winos-release`

- **类型**：Windows 发布仓库
- **分别是什么**：Windows 客户端打包、分发和发布自动化仓库。
- **更新信号**：Windows 安装包分发、发布包结构、最终外部版本号变化时，用于发布校验。
- **文档同步定位**：主要用于校验最终外部版本和安装包。
- **是否为主要文档事实源**：否，作为发布校验源

### 10. `heicodeDocs`

- **类型**：内部文档仓库
- **分别是什么**：内部项目文档仓库，仅输出说明文档和资料归档。
- **更新信号**：适合参考历史表述、目录结构和内部说明，但不适合作为唯一事实来源。
- **文档同步定位**：可参考结构与历史表述，但不应单独作为产品事实源。
- **是否为主要文档事实源**：否

### 11. `xmindlab-heicode.github.io`

- **类型**：对外文档仓库
- **分别是什么**：当前正式对外文档仓库，也是需要持续维护的文档仓库。
- **更新信号**：这里承接整理后的正式内容，不负责定义产品事实。
- **文档同步定位**：用于承接整理后的正式输出。
- **是否为主要文档事实源**：否

## 建议的同步顺序

1. 先确认本次变化属于平台、客户端、Agent 管理、蜂群还是 SK。
2. 找到对应主线仓库，读取 README、docs、集成说明、版本说明或最近改动。
3. 先更新公共页：`product/介绍.md`、`product/登录平台和使用平台指南.md`、`product/Agent构建/快速开始.md`、`product/版本更新.md`。
4. 再按专题更新团队使用、蜂群模式、权限安全、故障处理等专题页。
5. 最后统一检查页面之间是否存在术语不一致、流程不一致或过度承诺。

## 建议的日常维护方式

- 每次准备更新文档前，先判断本次变化来自平台、客户端、Agent 管理、蜂群还是 SK。
- 先读对应主线仓库的 README、docs、集成文档、版本说明或最近提交，再整理成用户可读表达。
- 先改公共页，再改专题页，最后统一检查术语、流程和限制说明是否一致。
- 如同一能力横跨多个仓库，应以主链路仓库为准，其他仓库只作为补充佐证。

## 建议优先维护的公共页

这些页面最容易被多个仓库同时影响，建议每次文档同步时优先检查：

- `product/介绍.md`
- `product/登录平台和使用平台指南.md`
- `product/Agent构建/快速开始.md`
- `product/版本更新.md`
- `product/隐私及法规/隐私政策.md`

如果这些公共页事实发生变化，再继续扩展到团队使用、蜂群模式、权限安全、故障处理等专题页。

## 维护时的快速判断规则

- 涉及平台入口、账号、资源绑定、模型余额、审计：优先看 `heicode-mananger`。
- 涉及客户端界面、任务视图、审批、更新、报错：优先看 `heicode-macos-release-dev` 与 `heicode-winos-release-dev`。
- 涉及单 Agent、Sub 模式、常驻 Agent、部署管理：优先看 `agent_management`。
- 涉及蜂群模式、审批链路、多租户运行时：优先看 `agent_swarm`。
- 涉及 SK 技能结构、技能权限、技能注册：优先看 `heicode-sk-hub`。
- 涉及安装包、公证、签名、最终发版：用 `heicode-macos-release`、`heicode-winos-release` 做发布校验。
- 涉及文案归档、内部表达参考：可参考 `heicodeDocs`，但不能把它当唯一事实源。

## 一句话结论

文档同步时，应把 `heicode-mananger`、`heicode-macos-release-dev`、`heicode-winos-release-dev`、`agent_management`、`agent_swarm`、`heicode-sk-hub` 视为主要事实源；发布仓库用于校验版本，文档仓库只负责承接整理后的正式输出。
