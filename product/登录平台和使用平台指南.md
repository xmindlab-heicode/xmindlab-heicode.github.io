# 登录平台和使用平台指南

## 面向用户

本文面向第一次使用 Heicode 的普通用户，说明如何登录、连接资源、下载客户端、启动 Agent 和查看结果。

## 访问入口

平台入口：
    
    
    https://code.heicode.cc
    

用户只需要登录 Heicode。普通用户不需要登录 CodeGW 后台，也不需要登录 Azure Key Vault。

## 第一步：登录 Heicode

  1. 打开 `https://code.heicode.cc`。

  2. 输入账号和密码，或使用平台支持的登录方式。

  3. 登录成功后进入 Heicode 控制台。

  4. 进入首页，查看资源、模型、任务和客户端下载状态。




登录后平台会读取：

  * 用户 ID。

  * 用户邮箱。

  * 用户角色。

  * `channelId`。




这些信息用于用户身份识别。其中 `channelId` 是跨平台身份标识，由 CodeGW（计费平台）侧用于额度与用量归集；Heicode Manager 不使用 channelId 覆盖本地模型分组（users.group），模型访问分组在 Manager 侧独立管理。

## 第二步：下载客户端

  1. 在 Heicode 控制台进入 `客户端下载`。

  2. 选择对应系统版本，例如 macOS 或 Windows。

  3. 下载并安装 Heicode 客户端。

  4. 打开客户端后，只需要登录 `https://code.heicode.cc`。




客户端不会要求选择模型提供方。客户端可用模型由 Heicode 提供。客户端登录后展示的可用模型列表完全来自 Heicode Manager 的 `/api/heicode/available-models` 接口（按用户分组/订阅在服务端收口），不使用本地预置（preset）模型列表，也不从 CodeGW 渠道后台读取。

macOS 提示说明：

  * 如果安装时出现系统安全提示，说明当前安装包可能还没有完成 Apple Developer ID 签名和 notarization。

  * 用户可以按平台提供的安装说明继续打开。

  * 长期方案应完成签名和 notarization。




## 第三步：在客户端输入想法

主要在 Heicode 客户端输入要做的事情。Manager 负责下载、资源准备清单、Agent 启动、状态和审计等辅助能力，不承担主开发对话。

例如：
    
    
    我想做一个面向小团队的任务管理 SaaS，
    需要登录、项目、任务、评论、通知和后台管理，
    希望部署到 Azure。
    

平台会生成：

  * 需求摘要。

  * 产品文档草案。

  * 推荐开发角色。

  * 需要绑定的资源。

  * 高危操作提示。




## 第四步：按准备清单连接资源

进入当前任务的 `准备清单`。这里不应该让用户面对大量参数，而是告诉用户还缺什么、下一步做什么。

可连接资源：

  * 代码仓库。

  * SK 仓库或技能包。

  * 项目文档。

  * 云账号。

  * 云资源。




推荐流程：
    
    
    连接 Git
    -> 连接项目文档或 SK
    -> 连接云账号
    -> 从云账号自动发现资源
    -> 确认本任务允许使用的资源范围
    

用户不需要把密钥贴到文档里。真实凭证由 Heicode 写入密钥保管器，Heicode 只保存 `secret_ref`。

## 第五步：确认系统推荐

Heicode 会自动生成推荐摘要，用户只需要确认是否合理。

摘要包括：

  * Product Agent。

  * Architect Agent。

  * Frontend Agent。

  * Backend Agent。

  * Reviewer Agent。

  * Ops Agent。

  * 建议使用的仓库、文档、SK 和云资源。

  * 高危操作规则。

  * 预计模型预算。




用户看到的是这种表达：
    
    
    本任务将使用 repo-main 和 aks-test。
    生产部署、生产数据库访问、删除云资源需要客户端审批。
    长期密钥不会下发给 Agent。
    

用户不需要查看完整后台授权、运行策略或模型参数。这些由 Heicode 在后台生成，并进入审计记录。

## 第六步：确认高危操作

高危操作会在客户端审批。

常见高危操作：

  * 生产部署。

  * 修改数据库。

  * 创建或删除云资源。

  * 访问生产密钥。

  * 大额模型预算消耗。




用户确认后，平台才会为子 Agent 派生短期凭证。

## 第七步：启动 Agent

用户确认摘要后，点击 `启动 Agent`。

Heicode 会在后台生成执行所需对象，包括子 Agent角色、AGENT.md、资源上下文、运行策略、模型预算和审批策略。

这些内容不作为普通用户步骤展示。Heicode 将它们提交给 Agent 平台执行，用户只看到启动结果、当前状态和需要处理的审批。

## 第八步：在客户端持续推进

任务开始后，主流程回到客户端。

闭环应理解为：
    
    
    客户端继续补充要求
    -> Heicode 生成下一步子任务
    -> Agent 执行需求/开发/测试/修复/部署中的当前环节
    -> Agent 按权限调用需要的 SK 工具
    -> 中间结果和失败原因回到客户端
    -> 用户继续修正或审批
    -> 最终返回交付物和部署结果
    

## 第九步：查看执行结果

用户可以在 Heicode 查看：

  * 当前任务状态。

  * 子 Agent 活动状态。

  * 运行日志。

  * 失败原因。

  * 模型用量。

  * 资源访问记录。

  * 高危审批记录。

  * 部署结果。




## 第十步：维护和升级

上线后，用户可以继续输入：

  * 修复 bug。

  * 增加功能。

  * 优化性能。

  * 升级依赖。

  * 调整部署资源。




Heicode 会复用已有项目上下文、资源连接和审计信息。

## 用户注意事项

  1. 不要把密钥写入需求、Markdown、聊天内容或 Git。

  2. 授权云资源时先从测试环境开始。

  3. 生产部署必须仔细确认审批内容。

  4. 如果不确定某个资源是否应该授权，选择只读或暂不授权。

  5. CodeGW 后台不是普通用户入口，模型和余额在 Heicode 中查看。
